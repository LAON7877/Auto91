# AutoTX 自動交易系統

**當前版本：v1.3.4** - 統一失敗通知格式與合約代碼顯示修復 (2025-07-04)

## 系統概述

AutoTX 是一個基於永豐 API 的期貨自動交易系統，支援大台指、小台指和微台指的智能交易，並整合了 ngrok 全自動管理系統和動態端口控制。

### 最新更新 (v1.3.4)
- **統一失敗通知格式**：實現所有訂單提交失敗訊息的統一格式處理
- **失敗訊息翻譯系統**：新增錯誤訊息中英文對照表，提供友善的錯誤訊息翻譯
- **合約代碼顯示修復**：修復手動平倉和訂單取消時顯示錯誤合約代碼的問題
- **提交類型判斷修復**：修正手動操作時提交類型顯示錯誤的問題
- **Python語法錯誤修復**：修復global宣告位置錯誤導致的語法錯誤

### 重要更新 (v1.3.3)
- **TG通知功能重大修復**：參考TXserver.py架構重構main.py，解決通知系統錯誤
- **ngrok自動化設置系統**：實現用戶自定義token設置、自動配置和驗證流程
- **Token記憶功能**：改用服務器端存儲，解決webview環境下的持久化問題
- **回調事件處理機制**：完整實現訂單回調處理，支援各種訂單狀態通知
- **代碼架構重構**：移除測試功能，簡化系統架構並提升穩定性

## 核心功能

- **智能交易日判斷**：支援期貨週六夜盤交易，正確處理民國年假期檔案
- **多合約交易**：支援 TXF（大台）、MXF（小台）、TMF（微台）
- **動態端口控制**：通過 `port.txt` 靈活配置系統端口
- **合約資訊管理**：一鍵重新整理合約資訊，包含代碼、交割日期、保證金等
- **ngrok 全自動管理**：自動啟動、重啟、關閉、升級，無需手動操作
- **Web 管理介面**：直觀的圖形化操作介面，包含 ngrok 狀態監控
- **Telegram 通知**：即時交易狀態通知
- **自動化交易**：支援策略自動執行
- **完整日誌系統**：前端系統日誌與 ngrok 請求日誌整合

## 系統架構

### 端口控制架構
- **統一配置**：使用 `port.txt` 作為端口設置的唯一來源
- **自動配置**：首次啟動時自動生成配置文件，預設端口 5000
- **錯誤處理**：完善的端口讀取錯誤處理和預設值機制

### 合約資訊管理
- **即時更新**：一鍵重新整理合約相關資訊
- **視覺反饋**：載入動畫提供更好的操作體驗
- **統一介面**：與其他功能區域保持一致的視覺和交互設計

### ngrok 自動管理架構
- **自動啟動**：程式啟動時自動啟動 ngrok 背景進程
- **自動重啟**：檢測到 ngrok 異常時自動重啟（5秒延遲）
- **自動關閉**：程式退出時自動清理 ngrok 進程
- **自動升級**：背景檢查並支援 ngrok 版本自動更新
- **錯誤恢復**：自動處理 4040 API 錯誤，確保服務穩定

### 交易日判斷架構
- **源頭**：`main.py` 中的 `is_trading_day_advanced()` 函數
- **調用鏈**：`web/main.js` → `/api/trading/status` → `main.py`
- **假期檔案**：支援民國年格式（如 `holidaySchedule_114.csv`）

### 交易時段
- **早盤**：08:45-13:45
- **夜盤**：14:50-次日05:00（週六夜盤到週六凌晨05:00）

### ngrok 管理特點
- **零配置**：用戶無需手動啟動或配置 ngrok
- **高可用性**：自動故障檢測和恢復機制
- **資源管理**：程式退出時自動清理，避免進程殘留
- **版本管理**：支援自動檢查和升級 ngrok 版本
