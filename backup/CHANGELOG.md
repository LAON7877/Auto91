# 版本變更紀錄

## v1.3.0 - 動態端口控制與合約資訊優化 (2025-07-01)

### 新增
- **動態端口控制**：新增 `port.txt` 作為端口設置的統一來源
- **合約資訊重新整理**：新增合約資訊一鍵重新整理功能
- **自動端口配置**：首次啟動時自動生成 `port.txt`，預設端口 5000

### 修改
- **端口控制重構**：所有端口設置統一從 `port.txt` 讀取
- **合約資訊介面**：優化合約資訊區域的按鈕樣式和交互
- **按鈕交互統一**：統一所有重新整理按鈕的樣式和動畫效果

### 技術細節
- **端口管理**：使用 `get_port()` 函數統一管理端口設置
- **錯誤處理**：完善端口讀取的錯誤處理和預設值機制
- **按鈕動畫**：使用 CSS 動畫實現按鈕載入效果
- **代碼複用**：統一使用 `refresh-account-btn` 樣式類

### 用戶體驗改進
- **靈活配置**：用戶可通過修改 `port.txt` 自由設置端口
- **一鍵更新**：合約資訊可通過按鈕快速更新
- **視覺反饋**：添加載入動畫提供更好的操作反饋
- **介面一致性**：保持所有功能區域的視覺和交互一致性

## v1.2.0 - ngrok 全自動管理系統重大更新 (2025-06-28)

### 新增
- **ngrok 全自動啟動**：程式啟動時自動啟動 ngrok，無需手動操作
- **ngrok 自動重啟機制**：當 ngrok 進程異常時自動重啟，確保服務持續可用
- **ngrok 自動關閉**：程式退出時自動關閉 ngrok 進程，避免殘留進程
- **ngrok 自動升級檢查**：背景檢查 ngrok 版本更新，支援自動升級
- **ngrok 4040 API 錯誤自動恢復**：處理 ngrok API 500 錯誤和編碼錯誤，自動重試
- **ngrok 請求日誌 API**：新增 `/api/ngrok/requests` 獲取外部請求日誌
- **ngrok 版本管理 API**：新增 `/api/ngrok/version`、`/api/ngrok/check_update`、`/api/ngrok/update`
- **ngrok 延遲和連接統計**：新增 `/api/ngrok/latency`、`/api/ngrok/connections`
- **程式退出清理機制**：確保所有資源正確釋放，包括 ngrok 進程

### 修改
- **ngrok 啟動流程**：改為背景進程啟動，不阻塞主程式
- **ngrok 狀態檢查**：增強狀態檢查邏輯，支援 offline 狀態自動重啟
- **ngrok 錯誤處理**：靜默處理 4040 API 錯誤，避免頻繁錯誤日誌
- **ngrok 進程管理**：使用 subprocess.Popen 管理 ngrok 進程生命週期
- **前端日誌系統**：新增 `/api/system_log` 接收前端系統日誌

### 修復
- **ngrok 進程殘留**：修復程式關閉時 ngrok 進程未正確終止的問題
- **ngrok 4040 API 超時**：修復 API 請求超時導致的狀態檢查失敗
- **ngrok 自動重啟循環**：修復重啟定時器可能重複觸發的問題
- **ngrok 版本檢查失敗**：修復版本檢查時的異常處理

### 技術細節
- **自動重啟機制**：使用 threading.Timer 實現 5 秒延遲重啟
- **進程管理**：subprocess.Popen 配合 terminate() 和 kill() 確保進程清理
- **API 錯誤處理**：try-catch 靜默處理 4040 API 的 500 錯誤和編碼錯誤
- **狀態檢查優化**：多次重試機制，避免單次失敗影響狀態判斷
- **清理函數**：cleanup_on_exit() 確保程式退出時完整清理

### 用戶體驗改進
- **零配置啟動**：用戶無需手動啟動或配置 ngrok
- **自動故障恢復**：ngrok 異常時自動恢復，無需人工干預
- **完整日誌記錄**：前端可查看 ngrok 請求日誌和系統狀態
- **版本自動更新**：支援 ngrok 版本自動檢查和升級

## v1.1.0 - 交易日判斷邏輯重大更新 (2025-06-28)

### 新增
- 修正期貨週六夜盤交易日判斷邏輯
- 統一交易日判斷源頭為 `main.py`
- 支援民國年假期檔案格式（如 `holidaySchedule_114.csv`）

### 修改
- **重大變更**：將週六視為交易日（因為期貨週六有夜盤到凌晨05:00）
- **架構重構**：`main.py` 成為交易日判斷的唯一源頭
- **檔案格式**：假期檔案使用民國年命名（西元年-1911）

### 修復
- 修正週六被錯誤判斷為非交易日的問題
- 移除對 `TXserver.py` 的依賴，避免重複實現
- 統一所有交易日判斷邏輯

### 技術細節
- `main.py` 中的 `is_trading_day_advanced()` 函數為唯一源頭
- 調用鏈：`web/main.js` → `/api/trading/status` → `main.py`
- 支援民國年假期檔案自動搜尋和解析
