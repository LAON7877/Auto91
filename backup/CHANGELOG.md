## v1.3.10 - 日誌顯示邏輯修正與格式優化 (2025-07-10)

### 重大修正
- **日誌方向顯示邏輯錯誤修正**：修正了平倉時方向顯示錯誤的嚴重問題
  - 原問題：平空倉（BUY+COVER）錯誤顯示為「多單」
  - 修正後：平空倉正確顯示為「空單」，平多倉正確顯示為「多單」
- **價格格式統一**：移除所有價格顯示中的 `$` 符號，統一格式

### 新增
- **修正後的日誌訊息生成邏輯**：
  - 開倉（NEW）：BUY=多單，SELL=空單
  - 平倉（COVER）：BUY=空單（平空倉），SELL=多單（平多倉）
- **Webhook回調機制完善**：確保webhook訊號生成完整的三步驟前端日誌
  - 步驟1：訊號接收 - `來自 webhook開倉訊號：開多`
  - 步驟2：處理開始 - `自動開倉：大台｜多單｜1 口｜22,000｜限價 (ROD)`
  - 步驟3：執行確認 - `開倉成功：大台｜多單｜1 口｜22,000｜限價 (ROD)`

### 修改
- **方向判斷邏輯重構**：
  ```python
  # 修正後的邏輯
  if str(octype).upper() == 'NEW':
      # 開倉：BUY=多單, SELL=空單
      if str(direction).upper() == 'BUY':
          direction_display = '多單'
      else:
          direction_display = '空單'
  elif str(octype).upper() == 'COVER':
      # 平倉：BUY=平空單, SELL=平多單
      if str(direction).upper() == 'BUY':
          direction_display = '空單'  # 平空單
      else:
          direction_display = '多單'  # 平多單
  ```
- **價格格式化統一**：
  - 移除所有 `$` 符號
  - 統一格式：`22,000` 而非 `$22,000`
  - 保證金顯示：`大台 150,000` 而非 `大台 $150,000`
- **損益顯示格式**：移除 `＄` 符號，統一為 `+20,000 TWD`

### 技術細節
- **測試案例覆蓋**：
  - 手動開多倉：`手動開倉：大台｜多單｜1 口｜22,000｜限價 (ROD)`
