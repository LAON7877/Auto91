## 系統特色
- **一鍵啟動**：雙擊 `快啟.bat` 即可啟動完整系統
- **動態端口控制**：通過 `port.txt` 自由配置系統端口
- **ngrok 全自動管理**：無需手動啟動、停止或配置 ngrok
- **自動故障恢復**：ngrok 異常時自動重啟，確保服務持續可用
- **完整日誌系統**：前端系統日誌與 ngrok 請求日誌整合

## 快速啟動步驟

### 1. 環境準備
```bash
# 下載並安裝 Python 3.8+ 與 pip
# 下載本專案並安裝依賴：
pip install -r requirements.txt
```

### 2. 端口配置（可選）
```bash
# 方法一：使用預設端口
# 首次啟動時系統會自動創建 port.txt，預設端口為 5000

# 方法二：手動配置端口
# 在專案根目錄創建或編輯 port.txt，填入想要使用的端口號
echo 5002 > port.txt
```

### 3. 配置設定
```bash
# 準備好永豐 API 憑證：
# - API Key
# - Secret Key  
# - 身份證字號
# - 憑證檔案 (.pfx)
# - 憑證密碼

# 準備 Telegram Bot：
# - Bot Token
# - Chat ID

# 準備 ngrok Token (v1.3.2 新要求)：
# - 註冊 ngrok 帳戶：https://ngrok.com/
# - 獲取 Authtoken（每個用戶需要自己的 token）
```

### 4. 系統自動初始化
啟動後系統會自動執行：
- ✅ 讀取或創建端口配置（port.txt）
- ✅ 啟動 Flask Web 服務器（使用配置的端口）
- ✅ 自動啟動 ngrok 背景進程
- ✅ 開啟 Web 管理介面
- ✅ 初始化交易日判斷系統

### 5. 登入與配置
1. 用瀏覽器開啟 http://localhost:{port}（port 為配置的端口號）
2. **設置 ngrok Token（v1.3.2 新增）**：
   - 點擊「設置」標籤頁
   - 點擊「管理 ngrok Token」按鈕
   - 輸入您的 ngrok Authtoken
   - 點擊「儲存並啟動」
   - 系統會自動驗證並啟動 ngrok tunnel
3. 填寫永豐 API 憑證和 Telegram Bot 設定
4. 上傳假期檔案和憑證檔案
5. 點擊「登入」開始使用

## ngrok 自動管理說明

### 無需手動操作
- **自動啟動**：程式啟動時自動啟動 ngrok
- **自動重啟**：檢測異常時自動重啟（5秒延遲）
- **自動關閉**：程式退出時自動清理 ngrok 進程
- **自動升級**：背景檢查並支援版本自動更新

### 狀態監控
- 前端可即時查看 ngrok 狀態
- 支援延遲、連接數、請求日誌監控
- 自動錯誤恢復，無需人工干預

## 系統功能

### 交易日判斷
- 支援期貨週六夜盤交易
- 自動讀取民國年假期檔案
- 統一交易日判斷邏輯

### 合約資訊管理
- 一鍵重新整理合約資訊
- 即時更新代碼、交割日期、保證金

### 交易管理
- 支援大台指、小台指、微台指
- 即時帳戶狀態和持倉監控

### 通知系統
- Telegram Bot 即時通知
- 前端系統日誌記錄

## 關閉系統

### 正常關閉
- 關閉 Web 介面視窗即可
- 系統會自動執行清理工作：
  - ✅ 停止 ngrok 進程
  - ✅ 登出永豐 API
  - ✅ 清理所有資源
  - ✅ 重置登入狀態

### 緊急關閉
- 使用 Ctrl+C 強制關閉
- 系統仍會嘗試執行清理工作

## 故障排除

### 常見問題
1. **端口被占用**：修改 port.txt 中的端口號後重啟系統
2. **ngrok 無法啟動**：檢查網路連線，系統會自動重試
3. **API 連線失敗**：檢查憑證設定和網路
4. **前端無法訪問**：確認配置的端口未被占用

### 自動恢復
- 大部分問題會通過自動恢復機制解決
- 如需手動干預，請參考 `backup/TROUBLESHOOTING.md`
