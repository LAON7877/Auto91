### 日誌顯示邏輯修正與格式優化維護
**重大修正維護**：

- **Webhook回調機制完善**：確保webhook訊號生成完整的三步驟前端日誌

**維護檢查清單**：
- [ ] 確認日誌方向顯示正確（平空倉顯示「空單」，平多倉顯示「多單」）
- [ ] 測試價格格式統一效果（移除所有 `$` 符號）
- [ ] 驗證Webhook訊號的完整三步驟日誌記錄
- [ ] 檢查向後相容性，確保舊功能正常運作
- [ ] 確認損益顯示格式統一（移除 `＄` 符號）
- [ ] 測試保證金顯示格式（移除 `$` 符號）

**方向顯示邏輯維護**：
- **開倉邏輯**：NEW + BUY = 多單，NEW + SELL = 空單
- **平倉邏輯**：COVER + BUY = 空單（平空倉），COVER + SELL = 多單（平多倉）
- **備援邏輯**：當octype未知時，BUY = 多單，SELL = 空單
- **測試案例**：確保所有交易組合的方向顯示正確

**價格格式統一維護**：
- **價格顯示**：`22,000` 而非 `$22,000`
- **保證金顯示**：`大台 150,000` 而非 `大台 $150,000`
- **損益顯示**：`+20,000 TWD` 而非 `＄20,000 TWD`
- **市價處理**：價格為0時顯示「市價」

**Webhook回調機制維護**：
- **步驟1**：訊號接收 - `來自 webhook開倉訊號：開多`
- **步驟2**：處理開始 - `自動開倉：大台｜多單｜1 口｜22,000｜限價 (ROD)`
- **步驟3**：執行確認 - `開倉成功：大台｜多單｜1 口｜22,000｜限價 (ROD)`
- **錯誤處理**：Webhook處理失敗時記錄錯誤日誌

**用戶體驗維護**：
- **日誌準確性**：確保所有交易日誌的方向顯示正確
- **格式一致性**：統一所有價格和損益的顯示格式
- **交易追蹤便利性**：正確的方向顯示讓用戶能準確追蹤交易歷史
- **Webhook透明度**：完整的三步驟日誌讓用戶清楚了解webhook處理流程

## ngrok 維護

### 自動管理機制
- **自動啟動**：程式啟動時自動啟動 ngrok 背景進程
- **自動重啟**：檢測到 ngrok 異常時自動重啟（5秒延遲）
- **自動關閉**：程式退出時自動清理 ngrok 進程
- **自動升級**：背景檢查並支援 ngrok 版本自動更新

### ngrok 版本升級

#### 自動升級流程
1. **背景檢查**：程式啟動時自動檢查 ngrok 版本
2. **版本比較**：與 GitHub 最新版本進行比較
3. **自動下載**：發現新版本時自動下載並更新
4. **備份機制**：更新前自動備份舊版本
5. **錯誤恢復**：更新失敗時自動還原備份

#### 手動升級流程
```bash
# 1. 檢查當前版本
curl http://localhost:5002/api/ngrok/version

# 2. 檢查更新
curl -X POST http://localhost:5002/api/ngrok/check_update

# 3. 執行更新
curl -X POST http://localhost:5002/api/ngrok/update
```

#### 升級注意事項
- 升級過程中 ngrok 會暫時停止服務
- 升級完成後會自動重新啟動
- 建議在非交易時段進行升級
- 升級前建議備份重要資料

## 交易日判斷維護

### 重要原則
- **唯一源頭**：交易日判斷邏輯只存在於 `main.py` 中
- **嚴禁重複**：不得在其他地方重複實現交易日判斷
- **統一調用**：所有功能都通過 `/api/trading/status` API 獲取交易日狀態

### 假期檔案維護
1. **檔案格式**：使用民國年命名（如 `holidaySchedule_114.csv`）
2. **檔案位置**：`server/holiday/` 目錄
3. **編碼格式**：Big5 編碼
4. **欄位格式**：
   ```csv
   日期,備註
   2025/01/01,非交易
   2025/01/02,o
   ```
   - `'o'` = 交易日
   - 其他或空白 = 非交易日
